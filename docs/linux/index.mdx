## Linux command starter pack

These are the commands you should know first if you want a practical Linux base.

### Navigation

* `pwd` - print the current directory
* `ls` - list files
* `ls -la` - show hidden files with details
* `cd /path/to/dir` - change directory
* `cd ..` - go up one level
* `cd ~` - go to your home directory

### Files and folders

* `touch file.txt` - create an empty file
* `mkdir folder` - create a directory
* `mkdir -p a/b/c` - create nested directories
* `cp source target` - copy files
* `mv source target` - move or rename files
* `rm file.txt` - remove a file
* `rm -r folder` - remove a directory recursively
* `cat file.txt` - print file contents
* `less file.txt` - view a file page by page

### Search and inspect

* `grep "text" file.txt` - search inside a file
* `find . -name "*.md"` - search by file name
* `which gcc` - find the binary path
* `whereis bash` - find related files
* `history` - show previous shell commands

### Permissions

* `chmod +x script.sh` - make a script executable
* `chmod 644 file.txt` - set common file permissions
* `chown user:group file.txt` - change ownership
* `id` - show your user and groups

### Processes and system info

* `ps aux` - list running processes
* `top` or `htop` - view live system usage
* `kill PID` - stop a process
* `uname -a` - show kernel and host details
* `whoami` - print your username
* `hostname` - print machine name

### Disk and memory

* `df -h` - show disk space usage
* `du -sh folder` - show folder size
* `free -h` - show memory usage

### Networking

* `ip a` - show IP addresses and interfaces
* `ip route` - show routing information
* `ping example.com` - test connectivity
* `ss -tulpn` - show listening sockets
* `curl https://example.com` - fetch a URL
* `ssh user@host` - connect to a remote machine
* `scp file user@host:/path/` - copy a file securely

### Package managers

* Debian/Ubuntu: `sudo apt update`, `sudo apt install package`
* Fedora/RHEL: `sudo dnf install package`
* Arch: `sudo pacman -S package`

### Bash one-liners

* `echo $HOME` - show an environment variable
* `export NAME=value` - set an environment variable
* `for f in *.cpp; do echo "$f"; done` - loop over files
* `command1 && command2` - run the second command only if the first succeeds
* `command1 || command2` - run the second command if the first fails

## Linux command table

| Area | Must-know commands | Why it matters |
| --- | --- | --- |
| Navigation | `pwd`, `ls`, `cd`, `tree` | Find your place in the file system fast |
| Files | `touch`, `mkdir`, `cp`, `mv`, `rm`, `cat`, `less` | Create, move, inspect, and clean up files |
| Search | `grep`, `find`, `which`, `history` | Locate text, files, and commands |
| Permissions | `chmod`, `chown`, `id` | Control access and understand ownership |
| Processes | `ps`, `top`, `htop`, `kill` | Inspect and manage running programs |
| System info | `uname`, `whoami`, `hostname`, `free`, `df`, `du` | Understand the machine state |
| Networking | `ip`, `ping`, `ss`, `curl`, `ssh`, `scp` | Work with remote machines and connectivity |
| Shell | `echo`, `export`, `for`, `&&`, `||` | Build useful command-line automation |

## A practical Linux workflow

1. Open a terminal.
1. Check where you are with `pwd`.
1. Inspect files with `ls -la`.
1. Edit or create files with your editor of choice.
1. Use `grep`, `find`, and `cat` to inspect data.
1. Use `chmod +x` for scripts and run them directly.
1. Use `ssh` and `scp` when you need remote access.
1. Use `top`, `df -h`, and `free -h` when you need system status.

## What a strong Linux learner can do

* Navigate the file system without hesitation
* Write simple Bash scripts for repeatable work
* Set environment variables correctly for builds and tools
* Move files safely between local and remote machines
* Check process, disk, and memory state when something looks wrong
* Read compiler logs and build output without getting lost

## Common mistakes to avoid

* Do not use `rm -r` without checking the path first.
* Do not confuse absolute paths like `/home/aitr` with relative paths like `docs/linux`.
* Do not forget `chmod +x` before running shell scripts directly.
* Do not assume environment variables persist unless you export them in the right shell file.
