This track is for readers who want a clean path into compiler engineering without getting buried under toolchain details too early.

## Who This Is For

- students learning compilers for the first time
- systems programmers who know C or C++ but not compiler architecture
- engineers who want to understand optimization pipelines before touching LLVM internals

## Outcome

By the end of this track, you should understand how a compiler is structured, why IR exists, how control flow and data flow shape optimization, and how a small compiler project fits together.

## Suggested Practice

- trace a simple source file through preprocessing, parsing, IR, and assembly
- identify one frontend responsibility and one backend responsibility for each compiler phase
- compare a hand-written optimization idea with what `-O2` or `-O3` already does
